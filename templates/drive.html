About

This project was made following the raising doubt about the big corporations as Google, Microsoft, etc. Its goal is to replicate the easy-to-navigate interface so that everyone can upload files and understand how to download files. It features a login with a .env file, API Requests and a node.JS backend.

Challenges

The main challenge was to setup the backend. I had never worked with apis, .env files or auth system. The security on this project is something to still take into account as I want it to be accesible online so that we can upload files from anywhere in the world. I also had to learn about linux with Linux Ubuntu Server, the fact that it has no UI, only a terminal, is really disturbing and I had to search some commands to modify files, display files, etc.

Solutions

I used a basic API based on a browser path that displays files in a JSON format and then stores them into the user's file. To create this user, I used a .env file and then checked if the user's folder existed. When the user connects on the drive, they only see their files on their subdivision with no admin access to guarantee privacy and trust that no one can see their files. Linux Ubuntu Server is a really well documented plateform so I had no trouble finding online what I needed to edit, delete and upload files. About security, as the project is still ongoing, I did not do a security checkup, but I always try to separate sensitive datas, as the website only request files when a user is connected and verified in the .env file. There is no possibility to create a user without having access to the .env file.

Future

This project is still ongoing, because it's local and it's based in France, I cannot work on it right now. I want to make it have the exact same features than Google One, and maybe integrate an LLM model to make it more responsive. This would be a great feature to have and it would let its users have their own AI without having to pay for Gemini.
